==============================================================================

WHY THIS FAILS ON FAB BUT NOT LOCALLY
------------------------------------------------------------------------------
Your Rider 'Development Editor' build uses shared/monolithic PCHs that pull in
nearly every engine header automatically. Fab builds with -Rocket against an
INSTALLED engine, enforcing strict Include-What-You-Use. Every type you rely on
implicitly must now be #included explicitly. That's the cause of essentially all
~900 errors below: missing includes and a handful of undeclared members — not
logic bugs.

FASTEST PATH: work the table below top-down. Fixing the top ~10 symbols
(UWorld, USkeletalMeshComponent, GEngine, UStaticMeshComponent, UPrimaryDataAsset,
FHitResult, FTimerManager, FPaths...) clears the large majority of errors, because
each missing base type cascades into dozens of downstream syntax errors like
"syntax error ')'", "'Super' is not a base", "missing ';'" — those vanish once the
real include is added. Don't chase the syntax errors directly.
